# 发起拼团功能 - 后端接口实现总结

## 📋 实现概述

本次实现了"发起拼团"功能的两个核心后端接口，允许用户自定义发起服务器拼团活动。

## 🔌 接口列表

### 接口 1：获取可发起的拼团活动列表

**接口路径**: `GET /pc/group/activity/initiable`

**请求参数**: 无

**响应示例**:
```json
{
  "code": 200,
  "msg": "查询成功",
  "data": [
    {
      "id": 1,
      "title": "AI服务器拼团 - 4核8G版",
      "cpu": "4核",
      "memory": "8GB",
      "storage": "100GB SSD",
      "basePrice": 2388.00,
      "defaultMinNum": 2,
      "defaultMaxNum": 5
    }
  ]
}
```

**实现位置**:
- Controller: `OshGroupServerController.initiableActivities()`
- Service: `IOshGroupServerService.selectInitiableActivities()`
- ServiceImpl: `OshGroupServerServiceImpl.selectInitiableActivities()`

---

### 接口 2：发起拼团（创建组团）

**接口路径**: `POST /pc/group/work/create`

**请求体**:
```json
{
  "activityId": 1,
  "minNum": 2,
  "maxNum": 5,
  "price": 2407.90,
  "duration": 12
}
```

**响应示例**:
```json
{
  "code": 200,
  "msg": "发起成功",
  "data": {
    "workId": 1001,
    "orderNo": "GRP20260430001234",
    "payUrl": null
  }
}
```

**实现位置**:
- Controller: `OshGroupServerController.create()`
- Service: `IOshGroupServerService.createGroupWork()`
- ServiceImpl: `OshGroupServerServiceImpl.createGroupWork()`

##  业务流程

### 发起拼团流程

```
1. 参数校验
   ├─ activityId 非空
   ├─ minNum ≥ 2
   ├─ maxNum ≥ minNum
   ├─ price > 0
   └─ duration 1-120

2. 活动校验
   ├─ 活动存在
   └─ 活动状态为进行中(status=1)

3. 权限校验（TODO）
   └─ 用户限购检查

4. 创建组团记录
   └─ 插入 osh_group_work 表

5. 创建拼团订单
   ├─ 生成订单号: GRP + 时间戳 + 随机数
   └─ 插入 osh_group_order 表

6. 返回结果
   ├─ workId (组团ID)
   └─ orderNo (订单号)
```

## ⚠️ 注意事项

### 1. 参数校验
- 前端已做基础验证，后端必须二次校验
- 所有数值范围校验都在 Service 层实现

### 2. 事务处理
- `createGroupWork()` 方法使用 `@Transactional` 注解
- 确保组团记录和订单记录的一致性

### 3. 订单号生成
- 格式: `GRP` + `yyyyMMddHHmmss` + `6位随机数`
- 示例: `GRP20260430153045123456`

### 4. 权限控制
- 两个接口都需要登录（非匿名接口）
- 用户ID从 ThreadLocal 获取

### 5. TODO 项
- 用户限购逻辑（per_user_limit）

- 支付链接生成（如需要立即支付）
