## 三、Controller 返回值封装类替换（AjaxResult → R\<T\>）

### 背景
统一使用项目中的 `R<T>` 泛型封装类替代 `AjaxResult`

### R\<T\> 常用方法

| 方法 | 说明 |
|------|------|
| `R.ok()` | 返回成功，无数据 |
| `R.ok(data)` | 返回成功，携带数据 |
| `R.ok(data, msg)` | 返回成功，携带数据和消息 |
| `R.fail(msg)` | 返回失败，携带错误消息 |

### 与 toAjax 结合使用
```java
// 新增/修改/删除操作推荐写法
public R<Void> add(@RequestBody OshCoures course) {
    return toAjax(oshCouresService.insertCourse(course)) ? R.ok() : R.fail("新增失败");
}
```

### 说明
- `toAjax()` 是 `BaseController` 提供的工具方法
- 入参为 service 返回的 `int`（影响行数），返回 `boolean`
- 影响行数 > 0 返回 `true`，否则返回 `false`

## 四、常见问题 & 解决方案

### 问题 1：`IOshCouresService` 无法解析

**原因**：`IOshCouresService.java` 文件缺少 `package` 声明

**解决**：在文件首行添加：
```java
package com.backstage.system.service;
```

**经验**：文件重命名或重新创建时，需检查 `package` 声明是否丢失。

---

### 问题 2：启动报错 `Failed to parse mapping resource: SysExcelSyncRecordMapper.xml`

**错误信息**：
```
Caused by: org.xml.sax.SAXParseException: 需要属性 "collection"，并且必须为元素类型 "foreach" 指定该属性。
```

**原因**：`target` 目录中存在编译遗留的 `SysExcelSyncRecordMapper.xml`，该文件源码中不存在，但 `foreach` 缺少 `collection` 属性。

**解决**：执行 Maven 清理重新编译：
```bash
mvn clean compile -DskipTests
```

**经验**：启动报 XML 解析错误时，优先排查 `target` 中是否存在脏文件，执行 `mvn clean` 解决。

## 五、Git 操作问题 & 解决方案

### 问题 1：部分文件未被提交（Service 实现类缺失）

**原因**：
- `SysCourseServiceImpl.java` 被手动删除后，`OshCouresServiceImpl.java` 是重新创建的
- Git 内容差异较大，无法自动识别为重命名，导致新文件未被跟踪

**解决**：
```bash
git add -A
git commit -m "提交信息"
```

**经验**：文件重命名后，内容变化过大时 Git 会识别为"删除 + 新增"，需手动 `git add` 新文件。

---

### 问题 2：提交中包含不需要的文件（FileUpload.vue）

**场景**：已提交但未推送的 commit 中包含不该提交的文件

**解决步骤**：
```bash
# 1. 撤销最近一次提交（保留文件变更）
git reset --soft HEAD~1

# 2. 将不需要的文件从暂存区移除
git reset HEAD backstage-ui/src/components/FileUpload/index.vue

# 3. 重新提交（不包含该文件）
git commit -m "新的提交信息"
```

**经验**：
- `git reset --soft HEAD~1`：撤销提交，文件变更保留在暂存区
- `git reset HEAD <file>`：将指定文件从暂存区移除，不影响文件本身
- 推送前务必使用 `git status` 确认待提交文件列表

## 六、开发规范总结

1. **类重命名时**：类名、文件名、package 声明、XML namespace、resultMap type/id 需全部同步修改
