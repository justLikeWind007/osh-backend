# 手动添加参团用户功能修复总结

## 📋 问题概述

**接口**：`GET /pc/group/work/detail?activityId={activityId}`  
**问题**：返回的 `users` 列表中缺少通过"手动添加参团用户"功能添加的用户数据

---

## 🔍 问题根源

### 核心问题：数据库字段名不匹配

**错误描述**：
- **XML 映射使用的字段名**：`created_time` 和 `updated_time`
- **数据库实际字段名**：`create_time` 和 `update_time`

**影响范围**：
- `insertGroupWork` 方法执行失败（SQL 语法错误）
- 手动添加用户时，`osh_group_work` 表无法插入记录
- 详情接口查询不到手动添加的用户

**发现过程**：
1. 检查 `addUserToGroup` 方法逻辑 → 确认已调用 `insertGroupWork`
2. 检查数据库 `osh_group_work` 表 → 确认没有手动添加的用户记录
3. 检查 XML 映射文件 → 发现字段名错误
4. 对比 SQL 建表语句 → 确认正确字段名为 `create_time` 和 `update_time`

---

### 数据流转验证

**手动添加用户完整流程**：

```
1. 管理员调用 POST /pc/group/user/add
   ↓
2. Service 层校验（活动状态、人数限制、重复参与等）
   ↓
3. 更新 osh_group_user_initiated.current_num + 1
   ↓
4. 在 osh_group_work 表插入参团记录 ✅ 现在可以正常插入
   ↓
5. 检查是否达到最低成团人数
   ↓
6. 如果已成团，更新状态为 group_status = 1
   ↓
7. 返回成功结果
```

**详情接口查询流程**：

```
1. 前端调用 GET /pc/group/work/detail?activityId={id}
   ↓
2. Service 层调用 selectGroupDetail
   ↓
3. 查询拼团活动基本信息
   ↓
4. 调用 selectGroupUsers(activityId)
   ↓
5. SQL 查询 osh_group_work 表 ✅ 现在可以查询到手动添加的用户
   ↓
6. LEFT JOIN osh_user 表获取用户信息
   ↓
7. 返回完整的 users 列表
```

## 🔧 数据库表结构参考

### osh_group_work 表（正确结构）

```sql
CREATE TABLE `osh_group_work` (
    `id`                    bigint          NOT NULL AUTO_INCREMENT,
    `group_activity_id`     bigint          NOT NULL,
    `user_id`               bigint          NOT NULL,
    `order_id`              bigint          DEFAULT NULL,
    `actual_price`          decimal(10,2)   NOT NULL,
    `remaining_months`      decimal(10,4)   DEFAULT NULL,
    `group_status`          tinyint         NOT NULL DEFAULT 0,
    `join_time`             datetime        DEFAULT NULL,
    `server_start_time`     datetime        DEFAULT NULL,
    `server_expire_time`    datetime        DEFAULT NULL,
    `create_time`           datetime        DEFAULT CURRENT_TIMESTAMP,        -- ✅ 注意字段名
    `update_time`           datetime        DEFAULT NULL ON UPDATE CURRENT_TIMESTAMP,  -- ✅ 注意字段名
    PRIMARY KEY (`id`),
    KEY `idx_group_activity_id` (`group_activity_id`),
    KEY `idx_user_id` (`user_id`),
    KEY `idx_join_time` (`join_time`)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4;
```

**参考 SQL 文件**：`sql/osh_server_group.sql`（第 61-78 行）

## ⚠️ 注意事项

1. **数据库表结构**：
   - 确保使用 `osh_server_group.sql` 中的表结构
   - 不要使用 `osh_group_module.sql` 中的旧表结构

2. **字段命名规范**：
   - 统一使用 `create_time` 和 `update_time`
   - 避免使用 `created_time` 和 `updated_time`

3. **事务管理**：
   - `addUserToGroup` 方法使用 `@Transactional` 确保原子性
   - 如果任何一步失败，整个事务回滚

4. **日志记录**：
   - 关键操作已记录日志
   - 可通过 `logs/sys-info.log` 查看操作记录
