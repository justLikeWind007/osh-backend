# 第三方 iframe 工具使用次数扣减设计

## 目标

工具模块中 `access_type = 2` 的工具通过 iframe 嵌入第三方页面。第三方页面本身不能直接扣减本系统的用户工具使用次数，原因是：

- 第三方页面不应该持有本系统用户 token。
- 扣次数必须走本系统后端，才能校验登录态、用户剩余次数、工具状态和权限。
- 第三方页面只负责告诉父页面“这次工具使用成功了”。

因此采用 `window.postMessage` 通信：

```text
第三方 iframe 工具使用成功
  -> iframe 页面发送 postMessage
  -> 本系统父页面监听 message
  -> 父页面校验来源、工具 ID、消息类型
  -> 父页面调用 /pc/tool/use/consume 扣减次数
```

## 当前父页面已实现的逻辑

前端工具列表页文件：

```text
osh-frontend/pages/tool/[page].vue
```

当前已经在页面挂载时监听 iframe 消息：

```js
onMounted(() => {
  loadTags();
  loadTools();
  loadRecommendTools();
  if (process.client) {
    window.addEventListener('message', handleIframeToolMessage);
  }
});

onBeforeUnmount(() => {
  if (process.client) {
    window.removeEventListener('message', handleIframeToolMessage);
  }
});
```

当前父页面只处理“正在展开的 iframe 工具”：

```js
const currentExpandedTool = computed(() => {
  return toolList.value.find((item) => item.isExpanded) || null;
});
```

收到消息后的校验逻辑如下：

```js
const handleIframeToolMessage = (event) => {
  const item = currentExpandedTool.value;
  if (!item || !isIframeTool(item)) {
    return;
  }

  const iframeOrigin = getUrlOrigin(getEmbedUrl(item));
  if (!iframeOrigin || event.origin !== iframeOrigin) {
    return;
  }

  const data = parseIframeMessageData(event.data);
  if (!data || data.type !== 'OSH_TOOL_USED') {
    return;
  }
  if (data.status && data.status !== 'SUCCESS') {
    return;
  }
  if (Number(data.toolId) !== Number(item.id)) {
    return;
  }

  handleToolUsed(item, data.usageKey);
};
```

也就是说，第三方页面发来的消息必须满足：

- 当前工具已经在页面上展开。
- 当前工具是 iframe 工具，也就是 `access_type = 2`。
- `event.origin` 必须等于当前工具 `iframe_url` 的 origin。
- `data.type` 必须等于 `OSH_TOOL_USED`。
- `data.status` 为空或等于 `SUCCESS`。
- `data.toolId` 必须等于当前展开工具的 ID。

校验通过后，父页面会调用已有扣次数逻辑：

```js
const handleToolUsed = async (item, usageKey) => {
  if (!item || !isPaidResourceType(item.resourceType)) {
    return;
  }
  const { message } = createDiscreteApi(['message']);
  if (consumingToolIds.value.has(item.id)) {
    return;
  }
  if (Number(item.remainingCount || 0) <= 0) {
    message.warning('工具使用次数不足，请先购买套餐');
    return;
  }
  setConsuming(item.id, true);
  try {
    const res = await apiConsumeToolUsage({
      toolId: item.id,
      usageKey: usageKey || buildUsageKey(item.id),
    });
    if (res?.code === 200) {
      item.remainingCount = Number(res.data ?? Math.max(0, Number(item.remainingCount || 0) - 1));
      item.purchasedFlag = item.remainingCount > 0 ? 1 : 0;
      item.totalUsage = Number(item.totalUsage || 0) + 1;
      message.success('已扣减 1 次工具使用次数');
    } else {
      message.error(res?.msg || '扣减工具使用次数失败');
    }
  } catch (e) {
    message.error(e?.data?.msg || e?.message || '扣减工具使用次数失败');
  } finally {
    setConsuming(item.id, false);
  }
};
```

注意：当前只对 `resourceType = CASH_ONLY` 或 `CASH_POINT` 的工具扣次数。免费、VIP、小班、内部资源不会扣次数。

## 第三方 iframe 页面需要发送的消息格式

第三方页面在“工具真实使用成功”后发送：

```js
window.parent.postMessage(
  {
    type: 'OSH_TOOL_USED',
    toolId: 123,
    usageKey: '123-唯一使用标识',
    status: 'SUCCESS'
  },
  'https://你的前端域名'
);
```

字段说明：

| 字段 | 是否必传 | 说明 |
| --- | --- | --- |
| `type` | 是 | 固定为 `OSH_TOOL_USED`，父页面只处理这个类型。 |
| `toolId` | 是 | 本系统工具 ID，必须和当前展开工具一致。 |
| `usageKey` | 建议传 | 本次使用的唯一标识，用于后续做后端幂等。 |
| `status` | 建议传 | 使用成功传 `SUCCESS`。失败时不要发送，或发送非 `SUCCESS`，父页面不会扣次数。 |

`postMessage` 的第二个参数不要写 `'*'`，生产环境必须写本系统前端域名。例如：

```js
window.parent.postMessage(message, 'https://osh.example.com');
```

开发环境可以临时写：

```js
window.parent.postMessage(message, 'http://localhost:3000');
```

## 第三方页面如何拿到 toolId

推荐 iframe 地址中拼上 `toolId` 参数。例如后台配置的 `iframe_url` 可以是：

```text
https://third-party.example.com/calculator?toolId=123
```

第三方页面读取：

```js
const params = new URLSearchParams(window.location.search);
const toolId = Number(params.get('toolId'));
```

如果第三方页面还需要知道父页面域名，也可以拼一个 `parentOrigin`：

```text
https://third-party.example.com/calculator?toolId=123&parentOrigin=https%3A%2F%2Fosh.example.com
```

第三方页面读取：

```js
const params = new URLSearchParams(window.location.search);
const toolId = Number(params.get('toolId'));
const parentOrigin = params.get('parentOrigin') || 'https://osh.example.com';
```

## 第三方原生 HTML 示例

下面是一个最小可用的第三方工具页面示例。用户点击计算按钮，计算成功后通知父页面扣次数。

```html
<!doctype html>
<html lang="zh-CN">
<head>
  <meta charset="UTF-8">
  <title>第三方计算器</title>
</head>
<body>
  <input id="left" type="number" value="1">
  <select id="operator">
    <option value="+">+</option>
    <option value="-">-</option>
    <option value="*">*</option>
    <option value="/">/</option>
  </select>
  <input id="right" type="number" value="2">
  <button id="calcBtn">计算</button>
  <div id="result"></div>

  <script>
    const params = new URLSearchParams(window.location.search);
    const toolId = Number(params.get('toolId'));
    const parentOrigin = params.get('parentOrigin') || 'https://osh.example.com';

    function buildUsageKey() {
      return `${toolId}-${Date.now()}-${Math.random().toString(16).slice(2)}`;
    }

    function notifyToolUsed() {
      if (!toolId) {
        console.warn('缺少 toolId，不能通知父页面扣次数');
        return;
      }

      window.parent.postMessage(
        {
          type: 'OSH_TOOL_USED',
          toolId,
          usageKey: buildUsageKey(),
          status: 'SUCCESS'
        },
        parentOrigin
      );
    }

    document.getElementById('calcBtn').addEventListener('click', () => {
      const left = Number(document.getElementById('left').value);
      const right = Number(document.getElementById('right').value);
      const operator = document.getElementById('operator').value;

      if (!Number.isFinite(left) || !Number.isFinite(right)) {
        alert('请输入有效数字');
        return;
      }
      if (operator === '/' && right === 0) {
        alert('除数不能为 0');
        return;
      }

      let result;
      if (operator === '+') result = left + right;
      if (operator === '-') result = left - right;
      if (operator === '*') result = left * right;
      if (operator === '/') result = left / right;

      document.getElementById('result').innerText = `结果：${result}`;

      // 只有工具真实执行成功后才发送消息。
      notifyToolUsed();
    });
  </script>
</body>
</html>
```

## 第三方 Vue 示例

如果第三方页面也是 Vue，可以这样写：

```vue
<template>
  <div>
    <input v-model.number="left" type="number">
    <select v-model="operator">
      <option value="+">+</option>
      <option value="-">-</option>
      <option value="*">*</option>
      <option value="/">/</option>
    </select>
    <input v-model.number="right" type="number">
    <button :disabled="calculating" @click="handleCalculate">
      {{ calculating ? '计算中' : '计算' }}
    </button>
    <div v-if="result !== null">结果：{{ result }}</div>
  </div>
</template>

<script setup>
import { ref } from 'vue';

const params = new URLSearchParams(window.location.search);
const toolId = Number(params.get('toolId'));
const parentOrigin = params.get('parentOrigin') || 'https://osh.example.com';

const left = ref(1);
const right = ref(2);
const operator = ref('+');
const result = ref(null);
const calculating = ref(false);

function buildUsageKey() {
  return `${toolId}-${Date.now()}-${Math.random().toString(16).slice(2)}`;
}

function notifyToolUsed() {
  window.parent.postMessage(
    {
      type: 'OSH_TOOL_USED',
      toolId,
      usageKey: buildUsageKey(),
      status: 'SUCCESS'
    },
    parentOrigin
  );
}

async function handleCalculate() {
  if (calculating.value) return;
  calculating.value = true;

  try {
    if (!Number.isFinite(left.value) || !Number.isFinite(right.value)) {
      alert('请输入有效数字');
      return;
    }
    if (operator.value === '/' && right.value === 0) {
      alert('除数不能为 0');
      return;
    }

    if (operator.value === '+') result.value = left.value + right.value;
    if (operator.value === '-') result.value = left.value - right.value;
    if (operator.value === '*') result.value = left.value * right.value;
    if (operator.value === '/') result.value = left.value / right.value;

    notifyToolUsed();
  } finally {
    setTimeout(() => {
      calculating.value = false;
    }, 800);
  }
}
</script>
```

## 有后端处理的第三方工具示例

很多第三方工具不是前端直接算，而是调用自己的服务端接口。此时必须在第三方接口返回成功后再通知父页面。

```js
async function handleGenerate() {
  if (submitting) return;
  submitting = true;

  try {
    const response = await fetch('/api/generate', {
      method: 'POST',
      headers: {
        'Content-Type': 'application/json'
      },
      body: JSON.stringify({
        text: inputText
      })
    });

    const result = await response.json();
    if (!response.ok || !result.success) {
      alert(result.message || '生成失败');
      return;
    }

    renderResult(result.data);

    window.parent.postMessage(
      {
        type: 'OSH_TOOL_USED',
        toolId,
        usageKey: `${toolId}-${result.requestId || Date.now()}`,
        status: 'SUCCESS'
      },
      parentOrigin
    );
  } finally {
    submitting = false;
  }
}
```

不要在请求开始前扣次数，也不要在接口失败时扣次数。只有第三方工具确认本次使用成功，才发送 `OSH_TOOL_USED`。

## 父页面安全校验说明

父页面不能只看消息内容，还必须校验来源。

当前实现中：

```js
const iframeOrigin = getUrlOrigin(getEmbedUrl(item));
if (!iframeOrigin || event.origin !== iframeOrigin) {
  return;
}
```

假设当前工具的 `iframe_url` 是：

```text
https://third-party.example.com/calculator?toolId=123
```

那么父页面只接受：

```text
event.origin === https://third-party.example.com
```

其它网站即使发送下面的消息，也不会被处理：

```js
window.parent.postMessage({
  type: 'OSH_TOOL_USED',
  toolId: 123,
  status: 'SUCCESS'
}, 'https://osh.example.com');
```

## 第三方页面防重复点击建议

第三方页面也应该防重复点击，避免短时间内发送多条扣次数消息。

最简单做法：

```js
let submitting = false;

async function handleUseTool() {
  if (submitting) return;
  submitting = true;

  try {
    const success = await doRealWork();
    if (success) {
      notifyToolUsed();
    }
  } finally {
    setTimeout(() => {
      submitting = false;
    }, 800);
  }
}
```

父页面目前也有防重复扣减：

```js
if (consumingToolIds.value.has(item.id)) {
  return;
}
```

但这个只能防住同一个页面瞬间重复触发，不能替代后端幂等。

## 后端幂等建议

当前前端请求扣次数时会传：

```js
{
  toolId: item.id,
  usageKey: usageKey || buildUsageKey(item.id)
}
```

建议后续增加工具使用流水表，例如：

```sql
create table osh_tool_usage_log (
  id bigint not null auto_increment comment '主键ID',
  user_id bigint not null comment '用户ID',
  tool_id bigint not null comment '工具ID',
  usage_key varchar(128) not null comment '本次使用唯一标识',
  consume_count int not null default 1 comment '扣减次数',
  status tinyint not null default 1 comment '状态：1-成功',
  create_by varchar(64) default null comment '创建人',
  create_time datetime not null default current_timestamp comment '创建时间',
  update_by varchar(64) default null comment '更新人',
  update_time datetime not null default current_timestamp on update current_timestamp comment '更新时间',
  delete_flag tinyint not null default 0 comment '删除标识：0-正常，1-删除',
  primary key (id),
  unique key uk_usage_key (usage_key)
) engine=InnoDB default charset=utf8mb4 collate=utf8mb4_0900_as_cs comment='工具使用次数扣减流水表';
```

后端扣次数逻辑建议：

```text
收到扣次数请求
  -> 校验 userId、toolId、usageKey
  -> 先插入 usage_log，usageKey 唯一
  -> 如果唯一键冲突，说明已经扣过，直接返回当前剩余次数
  -> 如果插入成功，再扣减用户可用次数
  -> 返回扣减后的 remainingCount
```

这样即使第三方 iframe 重复发送同一个 `usageKey`，也不会重复扣次数。

## 完整时序

```text
1. 用户在本系统打开工具列表页
2. 用户展开 access_type = 2 的 iframe 工具
3. 父页面加载 iframe_url
4. 用户在第三方工具中完成一次有效操作
5. 第三方页面执行 window.parent.postMessage
6. 父页面收到 message
7. 父页面校验 event.origin
8. 父页面校验 type/status/toolId
9. 父页面调用 /pc/tool/use/consume
10. 后端扣减用户该工具剩余使用次数
11. 父页面刷新当前工具 remainingCount
```

## 常见错误

### 错误 1：第三方页面直接调用本系统扣次数接口

不建议这样做。第三方页面不应该拿本系统 token，也不应该直接操作本系统用户资产。

### 错误 2：按钮一点击就发送扣次数消息

应该在工具使用成功后发送。如果用户点击后校验失败、接口失败、生成失败，都不应该扣次数。

### 错误 3：postMessage 第二个参数写星号

开发时可以临时使用：

```js
window.parent.postMessage(message, '*');
```

生产环境必须改成明确域名：

```js
window.parent.postMessage(message, 'https://osh.example.com');
```

### 错误 4：父页面不校验 event.origin

必须校验。否则任意网页都可能伪造消息让父页面扣次数。

### 错误 5：第三方页面传错 toolId

父页面会校验 `data.toolId` 是否等于当前展开工具 ID。传错不会扣次数。

## 推荐最终约定

第三方工具统一封装一个方法：

```js
function notifyOshToolUsed(options = {}) {
  const params = new URLSearchParams(window.location.search);
  const toolId = Number(options.toolId || params.get('toolId'));
  const parentOrigin = options.parentOrigin || params.get('parentOrigin') || 'https://osh.example.com';

  if (!toolId) {
    console.warn('notifyOshToolUsed 缺少 toolId');
    return;
  }

  window.parent.postMessage(
    {
      type: 'OSH_TOOL_USED',
      toolId,
      usageKey: options.usageKey || `${toolId}-${Date.now()}-${Math.random().toString(16).slice(2)}`,
      status: 'SUCCESS'
    },
    parentOrigin
  );
}
```

第三方工具每次真实使用成功后调用：

```js
notifyOshToolUsed();
```
