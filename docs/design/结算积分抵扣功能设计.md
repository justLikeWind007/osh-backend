# 结算积分抵扣功能设计

## Context

当前 `OrderServiceImpl.checkout()` 只支持全额现金支付或免费订单。需要在结算时支持用户选择使用积分抵扣，积分优先扣减后剩余部分走现金支付。兑换比例：1 积分 = 0.1 元。所有商品类型均适用。

## 设计要点

- 前端传 `usePoints=true` 时才扣积分，用户可选
- 积分扣减 + 订单创建在同一事务内，保证原子性
- 订单取消/支付失败时退还积分
- 参考 `ToolPurchaseServiceImpl` 中已有的积分扣减 + Redis 缓存刷新模式

#### 6.2 修改 checkout() 方法

在 `BigDecimal amount = money(reqVO.getPayableAmount())` 之后、`buildPendingOrder` 之前插入积分计算逻辑：

```java
// 积分抵扣计算
long pointsUsed = 0;
BigDecimal pointsDeductAmount = BigDecimal.ZERO;
BigDecimal actualPayAmount = amount;

if (Boolean.TRUE.equals(reqVO.getUsePoints())) {
    OshUserAsset userAsset = oshUserAssetMapper.selectById(reqVO.getUserId());
    if (userAsset != null && userAsset.getPoints() != null && userAsset.getPoints() > 0) {
        long availablePoints = userAsset.getPoints();
        // 最多可抵扣积分 = 应付金额 × 10（1元=10积分）
        long maxDeductiblePoints = amount.multiply(BigDecimal.TEN).longValue();
        pointsUsed = Math.min(availablePoints, maxDeductiblePoints);
        pointsDeductAmount = BigDecimal.valueOf(pointsUsed)
                .divide(BigDecimal.TEN, 2, RoundingMode.HALF_UP);
        actualPayAmount = amount.subtract(pointsDeductAmount);
    }
}
```

后续 `buildPendingOrder` 和 `buildPendingPayment` 使用 `actualPayAmount` 替代 `amount`，并设置 `order.setPointsAmount(pointsUsed)`。

#### 6.3 事务内扣减积分

private void createOrderPaymentAndDeductPoints(OshOrder order, OshPayment payment,
                                                Long userId, long pointsUsed) {
    executeInTransaction(() -> {
        orderMapper.insertOshOrder(order);
        payment.setOrderId(order.getId());
        paymentMapper.insertOshPayment(payment);

        if (pointsUsed > 0) {
            deductUserPoints(userId, pointsUsed, order.getOrderNo());

    if (pointsUsed > 0) {
        registerAssetCacheRefreshAfterCommit(userId);

#### 6.4 新增私有方法 — 扣减积分

```java
private void deductUserPoints(Long userId, long pointsToDeduct, String orderNo) {
    OshUserAsset userAsset = oshUserAssetMapper.selectById(userId);
    if (userAsset == null || userAsset.getPoints() == null || userAsset.getPoints() < pointsToDeduct) {
        throw new ServiceException("积分不足，请重试");
    }
    long beforeBalance = userAsset.getPoints();
    userAsset.setPoints(beforeBalance - pointsToDeduct);
    oshUserAssetMapper.updateById(userAsset);

    OshUserAssetRecord record = new OshUserAssetRecord();
    record.setUserId(userId);
    record.setChangeType(1); // 支出
    record.setChangeSource(3); // 购买商品
    record.setChangeAmount(pointsToDeduct);
    record.setBeforeBalance(beforeBalance);
    record.setAfterBalance(userAsset.getPoints());
    record.setRemark("订单【" + orderNo + "】积分抵扣");
    oshUserAssetRecordMapper.insert(record);
}
```

#### 6.5 新增私有方法 — 退还积分

```java
private void refundOrderPoints(String orderNo) {
    OshOrder order = orderMapper.selectByOrderNo(orderNo);
    if (order == null || order.getPointsAmount() == null || order.getPointsAmount() <= 0) {
        return;
    }
    OshUserAsset userAsset = oshUserAssetMapper.selectById(order.getUserId());
    if (userAsset == null) {
        log.error("退还积分失败，用户资产不存在，userId={}", order.getUserId());
        return;
    }
    long beforeBalance = Optional.ofNullable(userAsset.getPoints()).orElse(0L);
    long refundPoints = order.getPointsAmount();
    userAsset.setPoints(beforeBalance + refundPoints);
    oshUserAssetMapper.updateById(userAsset);

    OshUserAssetRecord record = new OshUserAssetRecord();
    record.setUserId(order.getUserId());
    record.setChangeType(0); // 收入
    record.setChangeSource(3); // 购买商品
    record.setChangeAmount(refundPoints);
    record.setBeforeBalance(beforeBalance);
    record.setAfterBalance(userAsset.getPoints());
    record.setRemark("取消订单【" + orderNo + "】退回积分");
    oshUserAssetRecordMapper.insert(record);

    registerAssetCacheRefreshAfterCommit(order.getUserId());
}
```

#### 6.8 Redis 缓存刷新

复用 `ToolPurchaseServiceImpl` 中的模式，在 `OrderServiceImpl` 中添加：

```java
private void registerAssetCacheRefreshAfterCommit(Long userId) {
    OshUserAsset asset = oshUserAssetMapper.selectById(userId);
    Long points = asset != null ? asset.getPoints() : 0L;
    if (!TransactionSynchronizationManager.isSynchronizationActive()) {
        refreshUserAssetCache(userId, points);
        return;
    }
    TransactionSynchronizationManager.registerSynchronization(new TransactionSynchronization() {
        @Override
        public void afterCommit() {
            refreshUserAssetCache(userId, points);
        }
    });
}

private void refreshUserAssetCache(Long userId, Long points) {
    Map<String, Object> userMap = redisCache.getCacheObject(OshUserConstants.LOGIN_USER + userId);
    if (userMap == null) return;
    Object assetObj = userMap.get(OshUserConstants.ASSET);
    if (!(assetObj instanceof Map)) return;
    Map<String, String> asset = (Map<String, String>) assetObj;
    asset.put(OshUserConstants.POINTS, String.valueOf(points));
    userMap.put(OshUserConstants.ASSET, asset);
    redisCache.setCacheObject(OshUserConstants.LOGIN_USER + userId, userMap);
}
```

## 并发安全

事务内 `selectById` + `updateById` 在 InnoDB 下会获取行锁，防止同一用户并发下单超扣。如果后续压测发现竞争严重，可升级为 `SELECT ... FOR UPDATE`（在 `OshUserAssetMapper` 加自定义方法）。

## 验证方式

1. `mvn compile` 确认编译通过
2. 手动执行 ALTER TABLE SQL
3. 测试场景：
   - usePoints=false → 行为与之前完全一致
   - usePoints=true，积分足够覆盖全额 → 走免费订单路径，积分扣减，资产流水正确
   - usePoints=true，积分部分覆盖 → actualPayAmount > 0，调用支付渠道，积分扣减正确
   - usePoints=true，积分为 0 → 等同于不使用积分
   - 订单取消 → 积分退还，Redis 缓存同步
   - 支付渠道创建失败 → 积分退还
