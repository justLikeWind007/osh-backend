# 手动添加参团用户功能调试与验证指南

## 📋 问题描述

`GET /pc/group/work/detail?activityId={activityId}` 接口返回的 `users` 列表中缺少通过"手动添加参团用户"功能添加的用户数据。

---

## 🔍 问题根源分析

### 已修复的问题

1. **字段名不匹配** ✅ 已修复
   - **问题**：XML 映射中使用 `created_time` 和 `updated_time`
   - **实际表结构**：`osh_group_work` 表中的字段名是 `create_time` 和 `update_time`
   - **影响**：手动添加用户时，`insertGroupWork` 方法因字段名错误导致插入失败
   - **修复位置**：`OshGroupServerMapper.xml` 第 241-242 行

2. **数据一致性保障** ✅ 已实现
   - 在 `addUserToGroup` 方法中同时在 `osh_group_work` 表创建参团记录
   - 更新 `osh_group_user_initiated.current_num` 字段
   - 检查是否达到最低成团人数并更新状态

## 🐛 故障排查

### 问题 1：手动添加用户后，osh_group_work 表中没有记录

**可能原因**：
1. 字段名不匹配（已修复）
2. 事务回滚
3. SQL 执行异常

**排查步骤**：

```bash
# 1. 查看应用日志
tail -f logs/sys-error.log | grep "addUserToGroup"

# 2. 检查是否有异常堆栈
grep -A 20 "addUserToGroup" logs/sys-error.log
```

**常见错误**：
- `Unknown column 'created_time'` → 字段名错误（已修复）
- `Cannot add or update a child row` → 外键约束问题
- `Data truncation` → 数据类型不匹配

---

### 问题 2：osh_group_work 表有记录，但详情接口返回空列表

**可能原因**：
1. `selectGroupUsers` SQL 查询条件错误
2. `group_activity_id` 关联不正确
3. 用户表联接失败

**排查步骤**：

```sql
-- 1. 直接执行查询 SQL
SELECT 
    u.id AS user_id,
    u.username,
    u.nickname,
    u.avatar,
    gw.join_time,
    IF(
        gw.id = (SELECT MIN(gw2.id) FROM osh_group_work gw2 WHERE gw2.group_activity_id = 1),
        TRUE,
        FALSE
    ) AS is_leader
FROM osh_group_work gw
LEFT JOIN osh_user u ON gw.user_id = u.id
WHERE gw.group_activity_id = 1
ORDER BY gw.join_time ASC;
```

**预期结果**：应该返回手动添加的用户记录

**如果 SQL 查询有结果但接口返回空**：
- 检查 MyBatis 的 `resultMap` 配置
- 检查参数传递是否正确

## 📊 数据一致性检查脚本

```sql
-- 检查数据一致性
SELECT 
    gui.id AS activity_id,
    gui.current_num AS initiated_current_num,
    COUNT(gw.id) AS work_record_count,
    gui.group_status AS initiated_status,
    CASE 
        WHEN gui.current_num = COUNT(gw.id) + 1 THEN '✅ 一致'
        ELSE '❌ 不一致'
    END AS consistency_check
FROM osh_group_user_initiated gui
LEFT JOIN osh_group_work gw ON gw.group_activity_id = gui.id
WHERE gui.delete_flag = 0
GROUP BY gui.id, gui.current_num, gui.group_status
HAVING consistency_check = '❌ 不一致';
```

**预期结果**：空集（表示所有数据一致）

### 核心修复点

1. **字段名修正**：确保 XML 映射中的字段名与数据库表结构完全一致
2. **事务保障**：`addUserToGroup` 方法使用 `@Transactional` 确保原子性
3. **双重写入**：同时更新 `osh_group_user_initiated` 和 `osh_group_work` 两张表
4. **防重复校验**：通过 `UNION ALL` 同时检查两张表

## 📝 注意事项

1. **数据库表结构**：确保使用 `osh_server_group.sql` 中的表结构，而非 `osh_group_module.sql`
2. **字段命名规范**：统一使用 `create_time` 和 `update_time`（非 `created_time` 和 `updated_time`）
3. **事务管理**：所有涉及多表更新的操作必须使用事务
4. **日志记录**：关键操作必须记录日志，便于排查问题
