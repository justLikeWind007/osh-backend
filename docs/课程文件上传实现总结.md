# 课程文件上传功能实现总结

## 一、实现概述

根据需求重新设计了课程管理系统的文件上传架构，核心原则是**职责分离**：
- ✅ 课程创建仅保存基础信息，不处理任何文件上传
- ✅ 文件上传通过独立接口在课程创建后执行
- ✅ 调用文件模块提供的统一上传接口（待开发）
- ✅ 保持事务一致性和完善的异常处理

---

## 二、已完成的接口

### 2.1 课程封面上传接口

#### API 定义
```http
POST /pc/course/cover/upload
Content-Type: multipart/form-data

参数:
- file: MultipartFile（必填）- 封面图片文件
- courseId: Long（必填）- 课程 ID
```

#### 实现功能
1. **文件校验**
   - 支持格式：bmp, gif, jpg, jpeg, png
   - 大小限制：5MB
   - 文件名长度限制：100 字符

2. **权限校验**
   - 仅课程创建者可上传封面
   - 校验课程是否存在

3. **数据更新**
   - 调用 `FileUploadUtils.upload(file, UploadPathEnum.COURSE_COVER)`（待实现）
   - 更新 `osh_coures.cover` 字段

#### 代码位置
- Controller: OshCouresController.uploadCourseCover()
- Service: CourseManageServiceImpl.uploadCourseCover()
- Service 接口：ICourseManageService.uploadCourseCover()

---

### 2.2 课时视频上传接口

#### API 定义
```http
POST /pc/course/section/video/upload
Content-Type: multipart/form-data

参数:
- file: MultipartFile（必填）- 视频文件
- courseId: Long（必填）- 课程 ID
- sectionId: Long（必填）- 章节 ID
```

#### 实现功能
1. **文件校验**
   - 支持格式：mp4, avi, mov, mkv, wmv, flv, webm
   - 大小限制：500MB

2. **业务校验**
   - 校验章节是否存在且属于该课程
   - 校验用户权限

3. **数据更新**
   - 调用 `FileUploadUtils.upload(file, UploadPathEnum.COURSE_VIDEO)`（待实现）
   - 更新 `osh_course_section.mediaUrl` 字段

4. **返回结果**
   ```json
   {
     "videoUrl": "/upload/course/video/123/2026/03/31/123456.mp4",
     "originalFileName": "lesson1.mp4",
     "fileSize": 52428800,
     "duration": 0,
     "videoCodec": "h264",
     "videoResolution": "1080p",
     "videoBitrate": 0
   }
   ```

#### 代码位置
- Controller: OshCouresController.uploadSectionVideo()
- Service: CourseManageServiceImpl.uploadSectionVideo(...)
- Service 接口：ICourseManageService.uploadSectionVideo(..., sectionId)

## 三、数据库表结构

### 3.1 现有表字段（无需修改）

#### osh_coures（课程主表）
- `cover` VARCHAR(500) - 存储课程封面图片 URL（一对一）
- `tryContent` TEXT - 可存储试看文档 URL（一对一）

#### osh_course_section（章节表）
- `mediaUrl` VARCHAR(500) - 存储课时视频 URL（一对多）
- `cover` VARCHAR(500) - 章节封面 URL（可选）

#### osh_course_material（资料表）
- `file_url` VARCHAR(500) - 存储资料文件 URL（一对多）
- `file_type` VARCHAR(50) - 文件类型
- `file_size` BIGINT - 文件大小

### 3.2 结论
✅ **无需新建关联表**,现有表结构已满足所有文件上传需求

## 六、使用示例

### 6.1 完整课程创建流程

```javascript
// 步骤 1: 创建课程（不含文件）
const courseId = await axios.post('/pc/course/create', {
    title: "Java 高级编程实战",
    type: "media",
    price: 99.00
});

// 步骤 2: 上传课程封面
const coverFormData = new FormData();
coverFormData.append('file', coverFileInput.files[0]);
coverFormData.append('courseId', courseId);
await axios.post('/pc/course/cover/upload', coverFormData);

// 步骤 3: 创建章节
const sectionId = await axios.post('/pc/course/section', {
    courseId: courseId,
    title: "第一章 Java 基础"
});

// 步骤 4: 上传课时视频
const videoFormData = new FormData();
videoFormData.append('file', videoFileInput.files[0]);
videoFormData.append('courseId', courseId);
videoFormData.append('sectionId', sectionId);
await axios.post('/pc/course/section/video/upload', videoFormData);

// 步骤 5: 上传课程资料
const materialFormData = new FormData();
materialFormData.append('file', materialFileInput.files[0]);
materialFormData.append('courseId', courseId);
materialFormData.append('materialName', '课后习题.zip');
await axios.post('/pc/course/material/upload', materialFormData);
```

## 七、异常处理机制

### 7.1 业务异常

| 异常场景 | 异常类型 | 用户提示 |
|---------|---------|---------|
| 课程不存在 | ServiceException | "课程不存在" |
| 章节不存在 | ServiceException | "章节不存在或不属于该课程" |
| 无权限操作 | ServiceException | "无权限上传该课程的封面" |
| 文件格式错误 | ServiceException | "仅支持{格式列表}" |
| 文件大小超限 | ServiceException | "{文件类型} 大小不能超过{X}MB" |
| 数据库更新失败 | ServiceException | "更新课程封面失败" |

### 7.2 事务回滚策略

所有上传方法都使用了 `@Transactional(rollbackFor = Exception.class)`:

```java
@Override
@Transactional(rollbackFor = Exception.class)
public void uploadCourseCover(MultipartFile file, Long courseId, Long userId) {
    // 1. 文件上传失败 → 自动回滚
    String url = FileUploadUtils.upload(...);
    
    // 2. 数据库更新失败 → 自动回滚
    courseMapper.updateCourse(...);
    
    // 3. 任何未捕获的异常 → 自动回滚
}
```

## 八、注意事项

1. **文件大小限制**:必须在枚举类中明确定义，避免硬编码
2. **文件类型校验**:由 FileUploadUtils.assertAllowed() 统一处理
3. **权限控制**:建议使用 @PreAuthorize 注解 + 自定义权限表达式
4. **并发上传**:同一课程的封面上传需要加锁，防止覆盖
5. **文件删除**:课程删除时应同步删除关联文件（定时任务或事件驱动）
6. **CDN 加速**:建议生产环境使用 CDN 加速视频和资料下载
